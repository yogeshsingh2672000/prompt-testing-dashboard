# Security Policy

## Reporting a vulnerability

Please do not open public GitHub issues for security vulnerabilities.

Instead:

1. Use GitHub security advisories if the repository has them enabled.
2. Share a private report with enough detail to reproduce the issue.
3. Include impact, affected routes or files, and any suggested mitigation if available.

## Scope

Please report issues related to:

- prompt execution or model request handling
- persistence and data exposure
- unsafe dependency behavior
- authentication or authorization if those layers are added later
- XSS, injection, or content-security issues in reports or rendering
